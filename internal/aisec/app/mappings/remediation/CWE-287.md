# CWE-287 ‚Äî Improper Authentication

**OWASP Top 10 2021:** A07:2021 ‚Äî Identification and Authentication Failures
**ASVS v4.0.3:** V2.1.1, V3.2.2, V3.3.1
**HIPAA:** ¬ß164.312(d) ‚Äî Person or entity authentication
**GDPR:** Art32(1)(b) ‚Äî Confidentiality and integrity

---

## What it is

Improper authentication means the application does not correctly verify that a user
is who they claim to be. Common failures:

- Accepting `alg: none` JWTs (no signature verification)
- Not checking token expiry (`exp` claim)
- Accepting tokens signed with a wrong key
- Missing claims (`sub`, `email`, `role`) leading to privilege escalation

In Pursh, improper auth means a patient can read another patient's records ‚Äî
a direct HIPAA ¬ß164.312(d) violation.

---

## Vulnerable code (JWT ‚Äî alg:none bypass)

```python
import jwt

# NEVER ‚Äî accepts unsigned tokens
payload = jwt.decode(token, options={"verify_signature": False})
```

An attacker crafts a token with `{"alg": "none"}` and arbitrary claims.

---

## Fixed code (RS256 with JWKS ‚Äî this repo's pattern)

```python
# From aisec/app/auth/oidc.py
from jose import jwt, JWTError

async def verify_token(token: str, jwks: dict) -> dict:
    payload = jwt.decode(
        token,
        jwks,
        algorithms=["RS256"],   # explicit ‚Äî never ["*"] or ["none"]
        options={"verify_exp": True},
    )
    for claim in ("sub", "email"):
        if claim not in payload:
            raise JWTError(f"Missing required claim: {claim}")
    return payload
```

## How Semgrep detects it

Rule: `python.jwt.security.jwt-none-alg.jwt-none-alg` (p/security-audit)
Also: `python.jwt.security.unverified-jwt-decode`

Semgrep flags `verify_signature: False` and `algorithms=["none"]` patterns.

## Fix checklist

- [ ] Always specify `algorithms=["RS256"]` ‚Äî never a wildcard
- [ ] Always set `verify_exp=True` ‚Äî never override
- [ ] Validate `sub`, `email`, `role` after decode; reject missing claims
- [ ] Cache JWKS with 300 s TTL and last-known-good fallback
- [ ] Test with an expired token ‚Äî must return 401
