# CWE-798 — Hardcoded Credentials

**OWASP Top 10 2021:** A07:2021 — Identification and Authentication Failures
**ASVS v4.0.3:** V2.10.1 (Credentials must not be hardcoded)
**HIPAA:** §164.312(d) — Person or entity authentication
**GDPR:** Art32(1)(a) — Pseudonymisation and encryption (keys must be protected)
**CIS Control:** 4.7 — Manage default accounts

---

## What it is

A hardcoded credential is any secret (API key, password, token, private key) embedded
as a string literal in source code. Because source code lives in git, the credential
is permanently in commit history — even after you "delete" it. Anyone with read access
to the repo (current or historical) can extract it.

For HIPAA: a hardcoded DB password or Supabase service-role key gives an attacker
direct read/write access to ePHI without any authentication bypass needed.

---

## Vulnerable code

```python
# NEVER — this commit will be in git history forever
SUPABASE_SERVICE_KEY = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJyb2xlIjoic2VydmljZV9yb2xlIn0.XXXX"
DATABASE_URL = "postgresql://aisec:SuperSecret123@prod-db.rds.amazonaws.com/aisec"
```

---

## Fixed code (AWS Secrets Manager)

```python
import boto3, json
from functools import lru_cache

@lru_cache(maxsize=1)
def get_db_url() -> str:
    client = boto3.client("secretsmanager", region_name="us-east-2")
    secret = client.get_secret_value(SecretId="prod/aisec/database-url")
    return json.loads(secret["SecretString"])["url"]
```

For local dev only: put secrets in `.env.local` (git-ignored). Never `.env`.

```bash
# .env.local (git-ignored — safe)
DATABASE_URL=postgresql+asyncpg://aisec:localpass@localhost:5432/aisec
```

## If a secret is already committed

1. Rotate the credential immediately — treat it as compromised.
2. Remove from code, commit the removal.
3. Rewrite git history: `git filter-repo` or BFG Repo Cleaner.
4. Force-push with `--force-with-lease` (coordinate with team).
5. Verify with `gitleaks detect --source . --no-git` that history is clean.

---

## Fix checklist

- [ ] Move secret to AWS Secrets Manager or Parameter Store
- [ ] Update code to call Secrets Manager at runtime (cached, not per-request)
- [ ] Add variable name to `.gitleaks.toml` allowlist only if it is a placeholder
- [ ] Run `git log -S "secret_value"` to confirm history is clean
