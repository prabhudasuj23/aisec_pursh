# CWE-502 — Deserialization of Untrusted Data

**OWASP Top 10 2021:** A08:2021 — Software and Data Integrity Failures
**ASVS v4.0.3:** V5.5.1 (Serialized objects from untrusted sources must be validated)
**HIPAA:** §164.312(c)(1) — Integrity (protect ePHI from improper alteration)
**GDPR:** Art32(1)(b) — Integrity of processing

---

## What it is

Unsafe deserialization occurs when an application reconstructs an object from
attacker-supplied data without validating its structure first. In Python:

- `pickle.loads(user_data)` — arbitrary code execution
- `yaml.load(user_data)` without `Loader=yaml.SafeLoader` — arbitrary code execution
- `eval()` / `exec()` on any external data

In AISec: SARIF documents are POST'd by CI pipelines. A compromised pipeline could
send a crafted payload. The SarifDocument Pydantic schema validator at the ingest
boundary is the primary defence.

---

## Vulnerable code

```python
import pickle, yaml

# NEVER — remote code execution
obj = pickle.loads(request.body)

# NEVER — yaml.load without SafeLoader is RCE
config = yaml.load(open("config.yaml"))
```

---

## Fixed code

```python
import yaml
from pydantic import BaseModel

# yaml — always use SafeLoader
config = yaml.safe_load(open("config.yaml"))

# For structured external data — validate with Pydantic first
class SarifDocument(BaseModel):
    version: str
    runs: list[SarifRun]

doc = SarifDocument.model_validate(raw_dict)  # raises ValidationError on bad input
```

## How Semgrep detects it

Rules:
- `python.lang.security.deserialization.avoid-pickle`
- `python.lang.security.audit.avoid-yaml-load.avoid-yaml-load`

Both are in the `p/python` and `p/security-audit` packs used by this repo.

---

## Fix checklist

- [ ] Replace all `pickle` usage with JSON + Pydantic validation
- [ ] Replace `yaml.load()` with `yaml.safe_load()` everywhere
- [ ] Never `eval()` or `exec()` data from any external source
- [ ] Validate all inbound structured data against a Pydantic model before processing
